# Wave Visualization Layout Design

**Date:** 2026-04-25

**Goal**

统一 Bohai 波场 PNG 与 GIF 的排版，使坐标轴、色标和标题规范、稳定、易读。

**Scope**

- 调整现有导出的图像版式
- 保留当前数据选择方式与颜色映射逻辑
- 不修改数据解析结构
- 不引入新的可视化后端

**Approved Direction**

- 使用标准科研图版式
- 保留完整坐标轴
- 坐标单位改为 `km`
- 去掉科学计数法偏移显示（例如 `1e6`）
- 右侧保留清晰竖直 `colorbar`
- PNG 与 GIF 使用完全一致的布局

**Layout**

- 主图居中，占据大部分版心
- 右侧单独预留固定宽度用于 `colorbar`
- 标题置于上方中央，格式为 `case | component | frame`
- 坐标轴标签固定为 `x (km)` 与 `y (km)`
- 主图保持等比例显示，避免拉伸

**Axis Rules**

- 原始坐标从米转换为千米后显示
- 坐标轴刻度使用普通十进制数字，不显示科学计数法整体偏移
- 刻度数量保持适中，避免遮挡

**Colorbar Rules**

- 固定在右侧竖直显示
- 标签使用当前分量名，例如 `Total`
- 数值范围在同一批次导图中保持一致
- `colorbar` 与主图之间保留稳定间距，避免贴边

**PNG Rules**

- 单张图片遵循统一版式
- 标题写明当前帧文件名信息
- 输出适合直接查看与汇报截图

**GIF Rules**

- 每一帧沿用与 PNG 相同的坐标轴、标题位置和 `colorbar`
- 仅更新标题中的帧号与主图数值
- 避免因自动布局导致帧间抖动

**Implementation Notes**

- 继续使用当前脚本结构
- 优先调整 Matplotlib 渲染版式
- 保留现有 PyVista 脚本，但以 Matplotlib 结果作为稳定输出基线

**Risks**

- 如果自动布局继续参与，GIF 可能出现帧间抖动
- 如果色条位置不固定，主图宽度可能不一致
- 如果仍保留原始米单位，刻度可读性会明显变差

**Validation**

- 导出 1 张 PNG，检查标题、坐标单位、`colorbar` 与边距
- 导出 1 个短 GIF，检查帧间是否抖动
- 导出完整 GIF，确认布局一致
